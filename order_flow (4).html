Orders data

Upload a CSV/XLS file or paste your Airtable orders export. Columns auto-detected (order date, order no., customer, mobile, SKU, order value, order status, etc). Re-importing updates existing rows by order no. + SKU and preserves any supplier or courier data already matched. If the order status column says "Cancelled", the order is marked Cancelled everywhere. Dispatch from office is derived automatically — it's true when order status says dispatched, or when any courier is assigned.

Click to upload orders file (CSV / TSV)

Auto-status logic: INSTOCK field = Yes / no supplier → Instock, not dispatched Supplier + bill no (dispatch date) → Received in WH Supplier assigned, no bill no → Pending supplier Sold Out → Stock out order status = Cancelled → Cancelled (overrides everything) order status = dispatched, or any courier assigned → Dispatch from office Order numbers in "All orders" always come from this Airtable import — courier/COD/return imports only update matching existing orders, never add new ones.

Supplier data

Upload a CSV file or paste your supplier export. Matched to existing orders by order no. If a SKU column is present, each row's status is applied only to that SKU (so a multi-SKU order shows a separate status per item); otherwise the status is applied to the whole order. Updates: supplier, instock, supplier status, supplier dispatch date, received in WH, dispatch from office. Import your orders first so there's something to match against.

Click to upload supplier file (CSV / TSV)

EWE (Easyway) courier data

Upload file or paste export from EWE portal. Matched to orders by Reference Number = order number. Updates: Courier Status EWE, Tracking ID EWE, Dispatch Date from Office EWE. If a Replaced column is present and checked for a row, that order is automatically logged as a return on the same order no.

Click to upload EWE file (CSV / TSV)

TFM courier data

Upload file or paste export from TFM portal. Matched by SHIPPER REF # = order number. Updates: Courier Status TFM, Tracking ID TFM, Dispatch Date from Office TFM. If a Replaced column is present and checked for a row, that order is automatically logged as a return on the same order no.

Click to upload TFM file (CSV / TSV)

COD status

Upload a CSV/XLS file or paste data with columns: order no, amount, received. Matched to existing orders by order no. Sets COD status to "Received" and records the COD amount received when the received column is marked (yes / y / true / 1 / x).

Click to upload COD file (CSV / XLS)

Returns

Upload a CSV/XLS file or paste data with columns: order no, received (whether the return has been received back or not). Matched to existing orders by order no.

Click to upload Returns file (CSV / XLS)
